
Deliverable D12 is produced in Step 4 and captures the **intrinsic capability bounds of the ego-vehicle** required to realise the requirement-based metrics defined in D11.

It defines what the system is *physically and computationally capable of*, independent of the environment in which it operates.

---

## Purpose

D12 establishes the **ego-side capability envelope**, providing conservative bounds for key system attributes such as:

- Braking performance  
- Perception timing  
- Decision and processing latency  
- Actuation response time  

These bounds are derived from:
- Vehicle specifications  
- System architecture  
- Design assumptions  

They represent the **minimum guaranteed performance** that can be relied upon during operation.

---

## Key Characteristics

- Expressed as **functional relationships**, not fixed values  
- Defined **independently of environmental conditions**  
- Conservative by design (i.e. worst-case or lower-bound assumptions)  
- Directly supports the evaluation of requirement-based metrics (D11)  


---

## Role in the Methodology

D12 forms one half of the metric composition in Step 4:

- **D12 (Ego):** What the system is capable of  
- **D13 (Environment):** What the route requires  

These are later combined in:
- **D14 (Unified Metrics)** → state-conditioned admissibility conditions  

This combination determines whether safe operation is possible under given conditions.

---

## Example

For braking capability, D12 might define:

- **Braking performance:**  
  A conservative lower-bound deceleration value based on system limits  

- **Perception timing:**  
  Maximum time required to detect and classify a signal  

- **Decision + actuation latency:**  
  Total delay between perception and braking command execution  

These may be expressed as relationships such as:

- Stopping distance as a function of speed and deceleration  
- Reaction distance as a function of speed and system latency  

Importantly, these values are defined **without reference to specific route conditions** (e.g. gradient, adhesion variability), which are instead introduced in D13.
